# Syndicate Architecture Rules

## 1. Character Classification Constraints
The Underdelve Syndicate uses a fixed-category system for architectural stability. **DO NOT** add new entries to these lists. Any new hero or enemy MUST be mapped to an existing category.

### **Strict Job Cap (11 Categories)**
All character jobs must be selected from the following list:
1.  **Swordsman**
2.  **Archer**
3.  **Priest**
4.  **Knight**
5.  **Thief**
6.  **Mage**
7.  **Berserker**
8.  **Guardian**
9.  **Scout**
10. **Paladin**
11. **Bard**

### **Strict Race Cap (10 Categories)**
All characters must be selected from the following list:
1.  **Human**
2.  **Elf**
3.  **Night-Elf**
4.  **Dwarf**
5.  **Orc**
6.  **Undead**
7.  **Beastman**
8.  **Demon**
9.  **Dragon**
10. **Titan**

## 2. Visual Style Guidelines
*   **Aspect Ratio**: Hero portraits must be **3:5 dimension**.
*   **Art Style**: JRPG Dark Fantasy / Gacha Anime style.
*   **Composition**: No frames, no text, and no UI elements within the image file itself.
*   **Signifiers**:
    *   **Bard Actions**: Use `note` icon (Music).
    *   **Debuff Actions**: Use `skull` icon (Skull).
    *   **Mage Actions**: Use `fire` or `zap`.
    *   **Heals**: Use `heart` or `sparkles`.

## 3. Core Combat Roles
All units must fulfill one of the four primary roles:
*   **TANK**: High HP/DEF, frontline protection.
*   **DPS**: Primary damage dealers.
*   **HEALER**: Focused on HP restoration.
*   **SUPPORT**: Focused on Buffs, Debuffs, and utility.
